Claude Code · Referência de Hooks

Hooks do Claude Code

Evento, quando dispara, onde criar e código pronto para usar.

exit 0
Sucesso. Claude Code lê o JSON do stdout e continua normalmente.
exit 2
Bloqueante. Stderr vai ao Claude como erro. A ação é impedida.
exit 1
Não-bloqueante. Aparece como aviso no transcript, execução continua.
início de sessão
SessionStart não bloqueia
Dispara toda vez que uma sessão começa (startup), é retomada (resume), após /clear ou após compactação.
quando usar

Injetar contexto inicial antes do primeiro prompt: branch atual do git, arquivos modificados, issue ativa, variáveis de ambiente do projeto.

onde criar
~/.claude/settings.json
.claude/settings.json

Crie o script em .claude/hooks/session-start.sh e dê chmod +x.

bash .claude/hooks/session-start.sh 
#!/bin/bash
BRANCH=$(git rev-parse --abbrev-ref HEAD 2>/dev/null)
CHANGED=$(git diff --name-only | head -5 | tr '\n' ', ')

jq -n \
  --arg b "$BRANCH" \
  --arg f "$CHANGED" \
  '{
    hookSpecificOutput: {
      hookEventName: "SessionStart",
      additionalContext: ("Branch: " + $b + "\nModificados: " + $f)
    }
  }'

exit 0
json .claude/settings.json 
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup",
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/session-start.sh"
          }
        ]
      }
    ]
  }
}
antes da tool executar
PreToolUse pode bloquear
Dispara antes de qualquer tool executar: Bash, Edit, Write, Read, Glob, Grep, mcp__*. Use matcher para filtrar qual tool.
quando usar

Bloquear comandos destrutivos (rm -rf, DROP TABLE). Validar que o Claude não vai mexer em arquivo proibido. Modificar o input da tool antes de rodar.

onde criar
.claude/settings.json

Use o campo if para filtrar subcomandos antes de spawnar o script — evita overhead desnecessário.

bash .claude/hooks/block-rm.sh 
#!/bin/bash
CMD=$(cat | jq -r '.tool_input.command')

if echo "$CMD" | grep -qE 'rm\s+-rf'; then
  jq -n '{
    hookSpecificOutput: {
      hookEventName: "PreToolUse",
      permissionDecision: "deny",
      permissionDecisionReason: "rm -rf bloqueado pelo hook de segurança."
    }
  }'
  exit 0
fi

exit 0
json .claude/settings.json 
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "if": "Bash(rm -rf *)",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/block-rm.sh"
          }
        ]
      }
    ]
  }
}
permissionDecision aceita: allow · deny · ask · defer. Não use "block" aqui — esse é o formato antigo, deprecado para PreToolUse.
após a tool executar
PostToolUse tool já rodou
Dispara logo após uma tool executar com sucesso. Recebe tool_input (o que o Claude mandou) e tool_response (o resultado). Use matcher para filtrar qual tool.
quando usar

Rodar lint/typecheck após edição de arquivo e injetar erros de volta no contexto. Log de auditoria de arquivos editados. Validar formato do output antes do Claude continuar.

onde criar
.claude/settings.json

additionalContext injeta texto no contexto do Claude. O Claude vê e corrige sem você pedir nada.

bash .claude/hooks/tsc-check.sh 
#!/bin/bash
FILE=$(cat | jq -r '.tool_input.file_path // empty')

# só roda para arquivos TypeScript
if [[ "$FILE" != *.ts && "$FILE" != *.tsx ]]; then
  exit 0
fi

OUT=$(npx tsc --noEmit 2>&1)

if [ $? -ne 0 ]; then
  jq -n --arg e "$OUT" \
  '{
    hookSpecificOutput: {
      hookEventName: "PostToolUse",
      additionalContext: ("Erros de tipo encontrados:\n" + $e)
    }
  }'
fi

exit 0
json .claude/settings.json 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/tsc-check.sh"
          }
        ]
      }
    ]
  }
}
PostToolUse + async: true background
Igual ao PostToolUse, mas o Claude não espera o script terminar. O resultado chega no próximo turno via systemMessage. Só disponível no tipo command.
quando usar

Quando a operação demora: rodar a suíte de testes completa, build da aplicação, deploy, análise estática pesada. Claude continua trabalhando enquanto isso roda.

onde criar
.claude/settings.json

Hooks async não podem bloquear. Use systemMessage no stdout para que o resultado apareça na interface do usuário.

bash .claude/hooks/run-tests-async.sh 
#!/bin/bash
FILE=$(cat | jq -r '.tool_input.file_path // empty')

if npm test > /dev/null 2>&1; then
  jq -n --arg f "$FILE" \
  '{"systemMessage": ("✓ Testes passaram após editar " + $f)}'
else
  jq -n --arg f "$FILE" \
  '{"systemMessage": ("✗ Testes falharam após editar " + $f)}'
fi
json .claude/settings.json 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/run-tests-async.sh",
            "async": true,
            "timeout": 120
          }
        ]
      }
    ]
  }
}
quando o claude termina de responder
Stop pode forçar continuar
Dispara quando o agente principal considera que terminou. Não dispara em interrupção do usuário. Verifique stop_hook_active no input para evitar loop infinito.
quando usar

Quality gate final: impedir o Claude de declarar que terminou se os testes estão falhando, se faltam arquivos esperados, ou se tem erros de lint. É a garantia mais forte de qualidade.

onde criar
.claude/settings.json

Retorne decision: "block" com reason obrigatório. O Claude recebe o motivo e continua trabalhando.

bash .claude/hooks/quality-gate.sh 
#!/bin/bash
INPUT=$(cat)

# evita loop: se já está em stop hook, não bloqueia
ACTIVE=$(echo "$INPUT" | jq -r '.stop_hook_active')
if [ "$ACTIVE" = "true" ]; then
  exit 0
fi

if ! npm test > /dev/null 2>&1; then
  jq -n '{
    "decision": "block",
    "reason": "Testes estão falhando. Corrija antes de finalizar."
  }'
  exit 0
fi

exit 0
json .claude/settings.json 
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/quality-gate.sh",
            "timeout": 60
          }
        ]
      }
    ]
  }
}
tipo prompt — sem script
type: "prompt" Haiku por padrão
Funciona em qualquer evento. Em vez de um script bash, o Claude Code chama um modelo LLM com seu prompt e o JSON do evento. O modelo responde com {"ok": true/false, "reason": "..."}.
quando usar

Avaliações em linguagem natural que seriam complexas de fazer em bash: "todas as tarefas foram concluídas?", "o código tem comentários em inglês?", "a resposta está no idioma correto?".

onde criar
.claude/settings.json

Use $ARGUMENTS no prompt para injetar o JSON do evento. Se ok: false, o reason vai como instrução para o Claude continuar.

json .claude/settings.json — Stop hook com avaliação por LLM 
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "prompt",
            "prompt": "Analise o contexto a seguir e verifique se TODAS as tarefas solicitadas pelo usuário foram concluídas. Contexto: $ARGUMENTS\n\nResponda APENAS com JSON válido:\n- {\"ok\": true} se tudo foi feito\n- {\"ok\": false, \"reason\": \"o que ainda falta\"} se algo ficou pendente",
            "timeout": 30
          }
        ]
      }
    ]
  }
}
notificações e eventos assíncronos
Notification side effect apenas
Dispara quando Claude Code envia uma notificação: permission_prompt (pediu permissão), idle_prompt (Claude ocioso), auth_success. Não pode bloquear — apenas side effects.
quando usar

Receber notificação no celular ou no sistema quando o Claude precisar de atenção, terminar uma tarefa longa, ou estiver aguardando permissão.

onde criar
~/.claude/settings.json

Normalmente vai na config global — você quer ser notificado em qualquer projeto, não só em um específico.

bash ~/.claude/hooks/notify.sh (macOS) 
#!/bin/bash
INPUT=$(cat)
MSG=$(echo "$INPUT" | jq -r '.message')
TYPE=$(echo "$INPUT" | jq -r '.notification_type')

osascript -e \
  "display notification \"$MSG\" with title \"Claude Code\" subtitle \"$TYPE\""

exit 0
json ~/.claude/settings.json 
{
  "hooks": {
    "Notification": [
      {
        "matcher": "permission_prompt|idle_prompt",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/notify.sh",
            "async": true
          }
        ]
      }
    ]
  }
}
SessionEnd não bloqueia
Dispara quando a sessão encerra: clear, resume, logout, prompt_input_exit, other. Timeout padrão de 1.5s — configure timeout no handler se precisar de mais.
quando usar

Salvar log da sessão, fazer commit automático das alterações, limpar arquivos temporários, ou enviar resumo do que foi feito por email/Slack.

onde criar
.claude/settings.json

Timeout padrão: 1.5s. Se precisar de mais tempo, defina timeout no handler. Máximo configurável: 60s.

bash .claude/hooks/session-end.sh 
#!/bin/bash
INPUT=$(cat)
REASON=$(echo "$INPUT" | jq -r '.reason')
SESSION=$(echo "$INPUT" | jq -r '.session_id')
STAMP=$(date +"%Y-%m-%d %H:%M")

echo "[$STAMP] sessão $SESSION encerrada: $REASON" \
  >> ~/.claude/session-log.txt

exit 0
json .claude/settings.json 
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/session-end.sh",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
